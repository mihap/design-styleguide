# 11. Do's and Don'ts

## Color Usage

### ✓ Do

- [Rule — grayscale as foundation]
- [Rule — accent reservation for interactive / key highlights]
- [Rule — semantic color consistency]
- [Rule — contrast minimum]
- [Rule — color blindness testing]

### ✗ Don't

- [Rule — accent on large background areas]
- [Rule — creating new colors outside palette]
- [Rule — misusing semantic colors]
- [Rule — low-contrast text]
- [Rule — mixing warm / cool grays]

---

## Spacing & Layout

### ✓ Do

- [Rule — grid adherence for all spacing decisions]
- [Rule — consistency within component types]
- [Rule — generous whitespace default]
- [Rule — tighter spacing for related elements]
- [Rule — larger gaps for distinct sections]

### ✗ Don't

- [Rule — arbitrary spacing values]
- [Rule — crowding elements]
- [Rule — inconsistent padding within similar components]
- [Rule — center-aligning body content / form labels]
- [Rule — stretching buttons to full width by default]

---

## Interactive Elements

### ✓ Do

- [Rule — visual feedback for all interactive states]
- [Rule — consistent focus rings for keyboard navigation]
- [Rule — minimum clickable area (touch target size)]
- [Rule — loading states for time-taking actions]
- [Rule — disabling buttons when action unavailable]

### ✗ Don't

- [Rule — removing focus outlines without alternative]
- [Rule — color-only state communication]
- [Rule — hiding interactive elements or making them too subtle]
- [Rule — hover-only interactions]
- [Rule — auto-submitting forms or triggering actions unexpectedly]

---

## Visual Hierarchy

### ✓ Do

- [Rule — hierarchy via size, weight, and color]
- [Rule — single primary button for most important action]
- [Rule — reduced prominence for secondary / tertiary actions]
- [Rule — whitespace for grouping and separation]
- [Rule — important content above the fold]

### ✗ Don't

- [Rule — making everything bold / emphasized]
- [Rule — competing focal points]
- [Rule — relying solely on color for hierarchy]
- [Rule — decorative elements that add visual noise]
- [Rule — UI competing with user content (forms, data)]

---

## Accessibility

### ✓ Do

- [Rule — WCAG contrast requirement]
- [Rule — text alternatives for icons / images]
- [Rule — keyboard navigation support]
- [Rule — semantic HTML (button, input, heading levels)]
- [Rule — screen reader + keyboard-only testing]

### ✗ Don't

- [Rule — placeholder text as only label]
- [Rule — relying on color alone to convey info]
- [Rule — keyboard focus trapping / loops]
- [Rule — minimum font size threshold]
- [Rule — disabling zoom or text scaling]
