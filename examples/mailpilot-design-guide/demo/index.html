MailPilot Design System Guidelines

A colorful Tailwind-oriented design system for SMB shared-mailbox automation, inspired by Notion's clarity and Stripe's precision.

01-design-philosophy.md

Design Philosophy

Core experience principles expressed as product UI cards.

Principle cards

Principles use restrained surfaces with colorful accents only where workflow meaning exists.

Context before automation

Show the message, sender, workflow, and next action before running rules.

Color means workflow state

Queue health, urgency, ownership, and automation status receive color before decoration does.

03-typography.md

Typography

Inter provides crisp scanning for inbox rows, setup flows, and automation explanations.

Primary and secondary font stacks

Inter is the main interface font; JetBrains Mono is the secondary utility font for code, rules, and message headers.

Primary sans

Inter handles mailbox lists, forms, buttons, and navigation.

Use for all product UI and prose.

Secondary mono

if sender.domain == 'vip.customer' then priority = high

Use for rule syntax, message headers, IDs, and snippets.

Type scale and main font size

The main app font size is Body at 0.875rem; 1rem is reserved for explanatory copy.

05-border-radius.md

Border Radius

Soft but restrained radius makes the system approachable without losing precision.

Radius scale in use

Controls use md, cards use lg, and pills use full radius.

Input and button radius

0.5rem keeps controls compact and clear.

Card radius

0.75rem separates panels without feeling playful.

Pill radius

Full radius is reserved for badges and filters.
