# 1. Design Philosophy

## Core Principles

MailPilot feels calm like a workspace, precise like infrastructure software, and colorful enough to make automation status easy to scan.

- **Context before automation** — Show the message, sender, workflow, and next action before asking users to trust automation.
- **One visible next step** — Highlight the recommended action and keep secondary actions quiet.
- **Color means workflow state** — Use color to identify queue health, urgency, ownership, and automation status.
- **Dense but breathable** — Support inbox-scale scanning while preserving enough whitespace for confidence.

---

## The MailPilot Prism Light Theme

Primary violet identifies automation and decisive actions without making the product feel cold.

Teal, amber, and coral accents make queues, status, and recommendations more legible while base surfaces stay workspace-neutral.

---

## Design Constraints

- Use primary violet for one dominant action per view.
- Use accent color only for workflow intelligence, not decoration.
- Keep inbox and table surfaces compact, but give setup and review flows more whitespace.
- Use motion only for state confirmation, loading, and panel entrance.
- Use Inter with tabular figures for counts, timestamps, and SLA metrics.
